NAME

citrun_check — check for rewrite activity

SYNOPSIS

citrun_check [dir]

DESCRIPTION

The citrun_check utility recursively searches for activity logs created by citrun_wrap(1).
When dir is given it is searched instead of the current working directory.
The first section output by citrun_check is a summary of all rewrite activity. It can contain any of the following events:
Source files used as input
The total number of C/C++ source files that had rewriting attempted on them.
Application link commands
Number of link commands detected and modified to include the runtime.
Rewrite successes
The number of times the rewriter successfully transformed the entire source file.
Rewrite failures
Rewriting a source file failed. There are two failure cases, either native parsing the original source file also fails, or, only citrun_inst failed parsing the source file.
Rewritten source compile successes
The number of times the rewritten source file is processed successfully by the native compiler.
Rewritten source compile failures
Compiling the rewritten source file failed. There are two failure cases, either compiling the original source file also fails or only compiling the rewritten source fails.

EXIT STATUS

The citrun_check utility exits 0 on success, and >0 if an error occurs.
