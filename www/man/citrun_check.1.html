NAME

citrun_check — check for rewrite activity

SYNOPSIS

citrun_check [-o output] [dir]

DESCRIPTION

The citrun_check utility recursively checks either dir, if given, or the current working directory for activity logs created by citrun_wrap(1).
If the -o option is specified then citrun_check will write to output instead of standard output.
The text below shows the output of citrun_check after citrun_wrap(1) had been used to instrument a simple project:
Summary: 
         1 Source files used as input 
         1 Application link commands 
         1 Rewrite successes 
         1 Rewritten source compile successes 
 
Totals: 
         6 Lines of source code 
        53 Milliseconds spent rewriting source 
         1 Function definitions 
         1 Return statement values 
         3 Total statements
The first section shows how thoroughly the project was instrumented, while the second section contains rewriting totals.

SUMMARY COUNTS

The most important summary events are failures. Source rewriting can fail for a few reasons and is not always a bad thing.
The following table lists all possible events and describes them:
“Source files used as input”
The total number of C/C++ source files that had rewriting attempted on them.
“Application link commands”
Number of link commands detected and modified to include the runtime.
“Rewrite successes”
The number of times the rewriter successfully transformed the entire source file.
“Rewrite failures”
Rewriting a source file failed. There are two failure cases, either native parsing the original source file also fails, or, only citrun_inst failed parsing the source file.
“Rewritten source compile successes”
The number of times the rewritten source file is processed successfully by the native compiler.
“Rewritten source compile failures”
Compiling the rewritten source file failed. There are two failure cases, either compiling the original source file also fails or only compiling the rewritten source fails.

EXIT STATUS

The citrun_check utility exits 0 on success, and >0 if an error occurs.
