CITRUN-WRAP(1) General Commands Manual CITRUN-WRAP(1)

NAME

citrun-wrap — instrument a build command

SYNOPSIS

citrun-wrap make | jam | gcc | cmake | ...

DESCRIPTION

Add instrumentation to any C/C++ source files compiled during a build command.
The build command must search PATH when finding compiler executables, which must be one of the following:
  • clang / clang++
  • egcc / eg++
  • gcc / g++
  • cc / c++
Because instrumentation is silent any activity is logged to the citrun.log file. Instrumentations summaries can be generated by the citrun-check(1) script.
Incremental building is supported. If the build command fails, fix the problem and then rerun the build command with citrun-wrap.

FILES

citrun.log
Instrumentation log.

EXIT STATUS

The citrun-wrap utility exits with the build command exit code.

EXAMPLES

Let “program” exist. It is:
  • written in C
  • compiled with gcc
  • built with make and make searches PATH for gcc
Prepending citrun-wrap to the build command adds instrumentation:
$ citrun-wrap make
Now see “program” run:
$ ./program
