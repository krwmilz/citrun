NAME

citrun-wrap — instrument a build command

SYNOPSIS

citrun-wrap build_cmd

DESCRIPTION

C It Run uses source code rewriting, also called instrumentation. The only time rewriting can succeed is at software compilation time.
The citrun-wrap utility enables rewriting on any source files compiled by build_cmd.
An important caveat is that build_cmd must search PATH when finding compiler executables, which must be one of the following:
  • clang
  • clang++
  • egcc
  • eg++
  • gcc
  • g++
  • cc
  • c++
Because the warnings and errors from the rewrite tool will look indiscernible from the warnings and errors generated by the native compiler, all rewriting activity is logged to the citrun.log file. Human readable summaries can be generated by the citrun-check(1) script.
Incremental building is supported, but mixing object files created by different versions of citrun-wrap is not.

FILES

citrun.log
Instrumentation log.

EXIT STATUS

The citrun-wrap utility exits with the build command exit code.

EXAMPLES

Let “program” be in a project that is:
  • written in C
  • compiled with gcc
  • built with make and make searches PATH for gcc
Prepending citrun-wrap to the projects build command adds instrumentation:
$ citrun-wrap make
Now see “program” run:
$ ./program
If no viewer is running then citrun-gl(1) is started.
