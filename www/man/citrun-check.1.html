NAME

citrun-check — check directories for log files

SYNOPSIS

citrun-check [dir]

DESCRIPTION

The citrun-check script recursively checks either dir, if given, or the current working directory for citrun.log files created by the rewrite tool.
When citrun-check does not have its stdout redirected it prints a “Checking” line that adds a single period for every file processed.
Once checking is complete two sections are printed. The text below shows the output of citrun-check after citrun-wrap(1) had been used to instrument a simple project:
Summary: 
         1 Source files used as input 
         1 Application link commands 
         1 Rewrite successes 
         1 Rewritten source compile successes 
 
Totals: 
         6 Lines of source code 
        53 Milliseconds spent rewriting source 
         1 Function definitions 
         1 Return statement values 
         3 Total statements
The first section is a summary regarding how thoroughly the project was instrumented. The most important counts are in this section.
The second section contains rewriting totals. These are the total number of visited source code elements from the source files used as input.

SUMMARY COUNTS

The most important counts are failures. Source rewriting can fail for a few reasons and is not always a bad thing.
The following table shows all of the possible counts, and describes them:
“Source files used as input”
The total number of C/C++ source files that had rewriting attempted on them.
“Application link commands”
Number of link commands detected and modified to include the runtime.
“Rewrite parse {warnings,errors}”
The total number of parsing problems the rewriter had.
“Rewrite successes”
The number of times the rewriter successfully transformed the entire source file.
“Rewrite failures”
Rewriting a source file failed. There are two failure cases, either native parsing the original source file also fails, or, only citrun-inst failed parsing the source file.
“Rewritten source compile successes”
The number of times the rewritten source file is processed successfully by the native compiler.
“Rewritten source compile failures”
Compiling the rewritten source file failed. There are two failure cases, either compiling the original source file also fails or only compiling the rewritten source fails.

EXIT STATUS

The citrun-check utility exits 0 on success, and >0 if an error occurs.
