NAME

citrun-check — check directories for log files

SYNOPSIS

citrun-check [dir]

DESCRIPTION

The citrun-check script recursively checks either dir, if given, or the current working directory for citrun.log files created by the rewrite tool.
When citrun-check does not have its stdout redirected it prints a “Checking” line that adds a single period for every file processed.
Once checking is complete two sections are printed. The text below shows the output of citrun-check after citrun-wrap(1) had been used to instrument a simple project:
Checking '/home/kyle/test' ..done 
 
Summary: 
         1 Calls to the rewrite tool 
         1 Source files used as input 
         1 Application link commands 
         1 Rewrite successes 
         1 Rewritten source compile successes 
 
Totals: 
         6 Lines of source code 
        53 Milliseconds spent rewriting source 
         1 Functions called 'main' 
         1 Function definitions 
         1 Return statement values 
         3 Total statements
The first section is a summary. It has counts that are related to source files and source compilation. The most important counts are in this section.
The second section contains rewriting totals. These counts are related to individual source code elements that were possibly rewritten during a previous citrun-wrap(1) session.

SUMMARY COUNTS

The summary section contains the most important counts. The following table describes each of them in detail:
“Calls to the rewrite tool”
The total number of times citrun-inst(1) was called.
“Source files used as input”
The total number of C/C++ source files that had rewriting attempted on them.
“Application link commands”
Number of link commands detected and modified to include the runtime.
“Rewrite parse {warnings,errors}”
The total number of parsing problems the rewriter had.
“Rewrite successes”
The number of times the rewriter successfully transformed the entire source file.
“Rewrite failures (False Positive)”
The number of times the rewriter encountered a fatal error however the native compiler also failed to compile the same code.
“Rewrite failures (True Positive!)”
The number of times the rewriter encountered a fatal error but the native compiler succeeded on the same code. This is bad.
“Rewritten source compile successes”
The number of times the rewritten source file is processed successfully by the native compiler.
“Rewritten source compile failures (False Positive)”
Similar to rewrite failure false positives, the number of times the native compiler fails on both the rewritten source and the original source.
“Rewritten source compile failures (True Positive!)”
The number of times the native compiler fails on the rewritten source and succeeds on the original source. This is bad.

EXIT STATUS

The citrun-check utility exits 0 on success, and >0 if an error occurs.
