NAME

citrun-inst — rewrite source code

SYNOPSIS

citrun-inst compile_cmd

DESCRIPTION

Warning: citrun-inst is not usually called directly. Use citrun-wrap(1) instead.
The citrun-inst utility performs source code rewriting. It visits statements in source files and adds extra code to some of them.
citrun-inst should accept any compile_cmd that the compilers listed in citrun-wrap(1) would accept.

CALLING DIRECTLY

When citrun-inst is called directly it only performs its source code rewriting step. That means it does not perform its usual call to compile the (possibly) rewritten source file.
Calling directly is useful for debugging rewrite failures and rewritten source compile failures.
In this mode, successfully rewritten source files have “.citrun” appended to their file name and diagnostic information is printed to both standard output and citrun.log.

EXIT STATUS

The citrun-inst utility exits 0 on success, and >0 if an error occurs.
